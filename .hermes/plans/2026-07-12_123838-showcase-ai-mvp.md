# showcase.ai MVP Implementation Plan

> **For Hermes:** Implement this plan incrementally with strict typed contracts, vertical tests, and small commits.

**Goal:** Build a Turborepo MVP that explores a SaaS application, plans an editable interactive walkthrough, stores observations and steps, and publishes a shareable demo.

**Architecture:** A Next.js App Router frontend calls a separate Express backend. The backend runs a small Pi-based orchestration pipeline composed of explorer, planner, and demo-generation agents. PostgreSQL/Prisma is the source of truth; screenshots are stored through a pluggable local/Vercel Blob adapter.

**Tech Stack:** Next.js, React, TypeScript, Tailwind CSS, Express, Zod, PostgreSQL, Prisma, Playwright, OpenAI Responses API, `@earendil-works/pi-agent-core`, Vercel Blob, Vitest, Turborepo.

## Key decisions

- **Asynchronous jobs:** exploration can take minutes; API requests enqueue work and return immediately.
- **Versioned observations:** agents communicate through stored records, not hidden prompt context, enabling future refresh/diff agents.
- **Credential isolation:** credentials are encrypted at rest, redacted from API responses, and deleted after terminal job states.
- **Safe exploration:** private/link-local targets are rejected by default, navigation is same-origin and bounded, and destructive-looking controls are never clicked automatically.
- **Storage abstraction:** local disk supports development; Vercel Blob supports deployment without changing agent code.
- **Fallback planning:** deterministic planning makes local development and tests useful without an OpenAI key; production uses Responses API structured output.

## Completion criteria

- A user can create a sample or real exploration job, inspect generated chapters and steps, edit them, publish, and open a public walkthrough URL.
- Data contracts and production builds pass strict TypeScript checks.
- The API and core planner/orchestrator behavior have automated tests.
- The application has documented Neon/OpenAI/Blob setup and explicit MVP safety boundaries.
